# Инструкция по входу под учетной записью администратора

## Вариант 1: Если пользователь `admin` есть в Keycloak

Если вы создали пользователя `admin` в Keycloak:

1. Откройте страницу входа: `http://localhost/login/`
2. Введите:
   - **Username**: `admin`
   - **Password**: пароль из Keycloak (который вы установили для пользователя `admin`)
3. Нажмите "Войти"

Система автоматически проверит учетные данные через Keycloak и авторизует вас.

## Вариант 2: Если пользователя `admin` нет в Keycloak (локальный пользователь Django)

Если пользователь `admin` существует только в Django (не в Keycloak):

1. Откройте страницу входа: `http://localhost/login/`
2. Введите:
   - **Username**: `admin`
   - **Password**: `admin123` (пароль по умолчанию из `.env` файла)
3. Нажмите "Войти"

Система попытается проверить через Keycloak, но если пользователя там нет, использует локальную аутентификацию Django.

## Проверка пароля администратора

Если вы забыли пароль, можно:

1. **Сбросить пароль через Django Admin** (если есть доступ):
   ```bash
   docker exec rubicon-api python manage.py changepassword admin
   ```

2. **Или создать нового суперпользователя**:
   ```bash
   docker exec rubicon-api python manage.py createsuperuser
   ```

## Рекомендация

**Лучше всего создать пользователя `admin` в Keycloak** для единообразной аутентификации:

1. Откройте Keycloak Admin Console: `https://static.88.68.91.77.ip.webhost1.net/admin/master/console/#/Rubik/users`
2. Создайте пользователя `admin` (если его еще нет)
3. Установите пароль для пользователя
4. Войдите на сайт с этими учетными данными

После входа пользователь `admin` автоматически получит права администратора в Django (если они уже были установлены).
