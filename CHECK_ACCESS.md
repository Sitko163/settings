# Проверка доступа к приложению

## Статус сервисов

✅ Все контейнеры запущены и работают
✅ Порт 8888 доступен на localhost
✅ Nginx отвечает на запросы
✅ Django API работает

## Адреса для доступа

**Основной адрес:**
- http://localhost:8888/admin/
- http://127.0.0.1:8888/admin/

**Учетные данные:**
- Логин: `admin`
- Пароль: `admin123`

## Если не открывается в браузере

1. **Попробуйте другой браузер** (Chrome, Firefox, Edge)

2. **Очистите кэш браузера** или откройте в режиме инкогнито

3. **Проверьте файрвол/антивирус** - они могут блокировать локальные подключения

4. **Попробуйте через IP WSL:**
   - Узнайте IP: `wsl hostname -I`
   - Используйте: `http://<WSL_IP>:8888/admin/`

5. **Проверьте через curl:**
   ```powershell
   curl.exe http://localhost:8888/admin/
   ```

6. **Проверьте логи:**
   ```bash
   wsl docker-compose logs webserver
   wsl docker-compose logs api
   ```

## Альтернативный доступ напрямую к API

Если nginx не работает, можно обратиться напрямую к API:
- http://localhost:8000/admin/ (если проброшен порт 8000)

Но для этого нужно добавить в docker-compose.yaml:
```yaml
api:
  ports:
    - "8000:8000"
```
