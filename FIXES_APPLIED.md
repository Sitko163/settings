# Исправления для стабильности приложения

## Проблемы, которые были исправлены:

1. **API контейнер перезапускался** - добавлен healthcheck для контроля готовности API
2. **Nginx подключался к API до его готовности** - добавлена зависимость webserver от api с условием healthcheck
3. **Отсутствующий файл logo.png** - создан placeholder файл
4. **Отсутствие wget в контейнере** - добавлен wget в Dockerfile для healthcheck

## Что было изменено:

### docker-compose.yaml
- Добавлен healthcheck для API контейнера
- Изменена зависимость webserver от api на `condition: service_healthy`

### api/Dockerfile
- Добавлен wget для healthcheck проверок

### Создан файл logo.png
- Создан placeholder для отсутствующего логотипа

## Текущий статус:

После перезапуска контейнеры должны работать стабильнее:
- API будет проверяться на готовность перед запуском Nginx
- Healthcheck будет контролировать состояние API
- Nginx не будет пытаться подключиться к недоступному API

## Если проблемы продолжаются:

1. Проверьте логи: `wsl docker-compose logs -f api`
2. Проверьте статус healthcheck: `wsl docker-compose ps`
3. Убедитесь, что все контейнеры в состоянии "healthy" или "Up"
